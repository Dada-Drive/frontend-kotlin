## Patterns établis pour Session B

1. **Aucun refactor de ViewModel** — UI-only en Session A. AuthViewModel, SessionViewModel, RoleViewModel, NameEntryViewModel intacts.
2. **Extraction stateless** quand le screen utilise `rememberLauncherForActivityResult` ou Hilt VM (cf. `OnboardingScreenContent` cas) → publique = wrapper, `internal = Content` testable.
3. **Snapshot de l'état error/loading** systématique quand un changement n'est visible que dans cet état (cf. `welcomeScreen_errorBanner_light`).
4. **Tokens v2 (R-4.5)** : `LocalAppColors.current.{accent, error, errorSoft, surfaceAlt, borderStrong, ...}` partout. Pas de `Color.X` hard-codé sauf `Color.White` / `Color.Black` (gardés là où le redesign le demande).
