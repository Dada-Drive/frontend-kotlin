## Architecture OCR

```
DriverSetupScreen (Compose, state holder)
    ├── LaunchedEffect(cinOcrFields)   → auto-fill cinNumber si blank
    ├── LaunchedEffect(permisOcrFields) → auto-fill suffix / expiry / categories si blank
    │
    ├── DriverPersonalStep (CIN)
    │      └── DocumentOcrCaptureSection (CIN front)
    │             └── TakePicture + FileProvider → File
    │                    └── onFileCaptured(file, bmp)
    │                           └── VM.uploadDocumentForOcr(file, OcrDocType.Cin)
    │
    └── DriverLicenseStep (Permis) — même pattern

DriverSetupViewModel
    ├── cinOcrProgress / cinOcrFields : StateFlow<…>
    ├── permisOcrProgress / permisOcrFields : StateFlow<…>
    └── uploadDocumentForOcr(file, docType) →
            UploadAndPollOcrUseCase(file, docType.wire)
                .collect { progress -> stateflow.update(progress) }

UploadAndPollOcrUseCase
    1. emit(Uploading)
    2. DocumentRepository.uploadForOcr(file, docType)        // POST multipart
    3. emit(Processing(uploaded))
    4. loop x30 every 2s :
         DocumentRepository.getOcrStatus(uploaded.id)        // GET poll
         - Ready  → emit(Ready) + return
         - Failed → emit(OcrFailed) + return
         - else   → emit(Processing) + continue
    5. emit(Timeout) si épuisement budget 60s

DocumentRepositoryImpl
    ├── uploadForOcr() — Retrofit @Multipart @Idempotent
    │       multipart : document (file) + docType (text)
    │       → ApiResponse<OcrUploadResponseDto> → domain map
    │
    └── getOcrStatus() — Retrofit GET /upload/document/ocr/:id
            → ApiResponse<OcrPollResponseDto> → domain map (OcrPollResult)
```

## Backend endpoints consommés (dada-api)

### POST `/upload/document/ocr`

- **Auth** : Bearer JWT obligatoire
- **Body** : multipart/form-data
  - `document` : binary (image JPEG full-res) — ⚠️ pas `file` (le brief original disait `file`)
  - `docType` : `cin|permis|carte_grise|assurance`
- **Réponse 201** :
  ```json
  {
    "success": true,
    "data": {
      "document": { "id": "...", "url": "...", "key": "...", "docType": "cin", "status": "queued", "adminStatus": "pending" }
    }
  }
  ```

### GET `/upload/document/ocr/:id`

- **Réponse 200** (shape **flat**, pas de wrapper `document`) :
  ```json
  {
    "success": true,
    "data": {
      "documentId": "...",
      "docType": "cin",
      "status": "ok" | "review" | "failed" | null,
      "adminStatus": "pending|approved|rejected",
      "fields": null | { "nom", "prenom", "numero_cin", "date_naissance", ... },
      "meta": null | { "confidence", "status", "processing_time_ms" },
      "rejectionReason": null | "..."
    }
  }
  ```
  - `status=null` ⇒ encore queued/processing (worker pas terminé)
  - Mappé en `OcrLifecycle.Queued|Ready|Failed` côté domaine

## OcrLifecycle mapping

| Wire status (backend) | Domain `OcrLifecycle` |
|---|---|
| `queued`, `null` | `Queued` |
| `processing` | `Processing` |
| `ok`, `review` | `Ready` |
| `failed` | `Failed` |

## Tokens Option A — auto-fill rules

| Doc | OCR field | Auto-fill cible | Guard |
|---|---|---|---|
| CIN | `numero_cin` | `cinNumber` (state) | `cinNumber.isBlank()` |
| Permis | `numero` | `licenseSuffix` (strip `TN-`) | `licenseSuffix.isBlank()` |
| Permis | `date_expiration` (ISO) | `licenseExpiryInput` (DDMMYYYY) | `licenseExpiryInput.isBlank()` |
| Permis | `categories` | `licenseCategories` (Set<Char>) | `licenseCategories == setOf('B')` (rest user-untouched) |

Tous les champs auto-fillables conservent la priorité utilisateur : si le driver a tapé quelque chose, OCR ne l'écrase pas.
